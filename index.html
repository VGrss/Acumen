01 — The Problem

AI product work has a slop problem.

Ask any LLM to write a PRD, build personas, or draft a strategy. You'll get the same predictable output every time:

  • — Vague value props that could describe any product in any market
  • — Demographic personas like "Marketing Manager, 28-35" instead of behavioral segments
  • — Vanity metrics that always go up and never tell you if something worked
  • — Feature checklists with no thesis, no sequencing, no opinion
  • — Specs that could belong to any company — every section filled, none specific
  • — Hedge documents where every recommendation is qualified into meaninglessness

Acumen fights this bias at the source. Not with templates — with vocabulary.

03 — Commands

15 commands. A shared vocabulary between you and your AI.

Each command is a focused product operation — not a template, but a thinking posture that pushes back when the output smells generic.

Context Layer — build & maintain product knowledge
/teach-acumenOne-time setup that gathers product context and saves it to .acumen.md. Run once to establish persistent product guidelines.
/scoutBuild and maintain a living competitor map. Supports both quick scans and deep competitive analysis.
/personaBuild and maintain behavioral personas grounded in real user patterns.
/featuresBuild and maintain a lightweight feature inventory. Use after shipping, when planning, or to understand the current product surface.
/update-acumenFetch the latest Acumen skills from GitHub and update local copies.
Audit Layer — assess product health
/orientationAudit product identity — what we do, for whom, whether the feature surface is coherent with the thesis.
/defensibilityAudit what’s hard to copy — moats, switching costs, data advantages, network effects, and AI-era resilience.
Ideate Layer — find problems and opportunities
/diagnoseFind current problems in the product based on data, value delivery for main personas, and current features. Suggests /workshop for ideation on solutions.
/measureCheck KPI health — what’s working, what’s not, where to dig deeper. Suggests /workshop for opportunities.
/workshopIdeation workshop to explore opportunities — value chain analysis, competitive gaps, user feedback synthesis.
Craft Layer — build product artifacts
/roadmapPlan a sequence of bets, break into shippable increments, prioritize.
/incrementScope a product increment end-to-end — context, spec, implementation, wireframes, and engineering considerations.
/critique-productEvaluate and validate a PM artifact — score for rigor, test riskiest assumptions, and decide if it’s ready to ship.
Communicate Layer — share with the world
/narrateWrite product communication for a specific audience, including when stakes are high and stakeholders disagree.
/changelogWrite a changelog from recent PRs and commits, grounded in feature context.

06 — Get Started

Install in under a minute.

Acumen follows the Agent Skills spec. Drop the skills into your AI tool and go.

Recommended

# Auto-detect your tool and install
npx skills add contenty/acumen

Works with any tool that supports the Agent Skills spec.

Claude Code (project)

# Copy skills into your project
cp -r .claude your-project/
cp -r .agents your-project/

Skills load automatically via Claude Code's skill system.

Claude Code (global)

# Install skills globally
mkdir -p ~/.claude/skills
cp -r .agents/skills/* ~/.claude/skills/

Available in every project on your machine.

Other AI Tools

# Copy the Agent Skills directory
cp -r .agents/skills/ ~/your-tool/skills/

Any tool supporting the Agent Skills spec.

Then run setup

# One-time product context
/teach-acumen

# Start using commands
/diagnose
/orientation
/measure

Works with: Claude Code · Cursor · Gemini CLI · Codex CLI · VS Code Copilot · Kiro · OpenCode · and any tool supporting the Agent Skills spec.

07 — Principles

Design decisions.

  • Mindset injection > workflow templates
    Skills set a thinking posture, not process steps.
  • Opinion > comprehensiveness
    "RICE doesn't ship products, judgment does."
  • Brevity > coverage
    50-100 lines per skill. Depth lives in reference docs.
  • Context layer feeds all other layers
    Not passive storage — active inputs that shape every artifact.
  • Vocabulary calibration
    Replace AI slop with product-fluent language.
  • Natural flow between skills
    Skills suggest the next skill: /diagnose → /workshop → /increment → /critique-product.

FAQ

How do I install Acumen?
Run npx skills add contenty/acumen for auto-detection, or copy the .agents and .claude directories into your project root.
How do I update?
Run /update-acumen from within your AI tool, or run npx skills update. Your .acumen.md and context files are preserved.
Commands aren't showing up.
Verify the skill files are in the right directory for your tool. For Claude Code, check that .claude/skills/ contains the symlinks. For Cursor, ensure Agent Skills is enabled.
Which tools are supported?
Claude Code, Cursor, Gemini CLI, Codex CLI, VS Code Copilot, Kiro, OpenCode, and any tool supporting the Agent Skills spec.
Is Acumen free?
Everything is Apache 2.0: skills, commands, reference docs — all free and open source.
Do I need /teach-acumen in every project?
Yes, once per project. It generates .acumen.md with your product's context. Without it, commands still work but produce less targeted output.
